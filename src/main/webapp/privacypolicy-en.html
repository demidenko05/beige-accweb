Beigesoft™ Enterprise Information System. Privacy Policy.

Beigesoft™ EIS all versions.

Beigesoft™ EIS stores your information in database files. Log files may also contain private information.

If you enable the "Show debugging messages" setting, then the log files logStd0.log and logStd1.log will contain private information. When you are going to transfer these Log files to a third party (for example, to the programmer serving this software), then you need to delete all private information from these files.

Beigesoft™ EIS is based on the WEB interface, you need to use the latest version of a reliable Web Browser. You better refuse to install any extensions or plug-ins for the Web browser.

Beigesoft™ EIS standalone versions.

These versions are based on the built-in private A-Jetty Web server. This private Web Server is not accessible outside of your computer or Android device. These versions require authorization with a strong password (you create a password yourself, and the software controls its reliability). These versions allow a maximum of 6 failed authorization attempts. Access will be blocked for 15 minutes in case of 6 unsuccessful login attempts. You need to periodically check the logSec.log file for authorization information. These versions use modern encryption of data passing from the server to the browser (HTTPS connection) and exchanged files (databases and log files). These versions allow you to work with several databases, you can create, switch, delete and encrypt databases by using the Web interface. You can use encryption (Web interface) for a transferred database.

Beigesoft™ EIS on embedded A-Jetty for a computer.

Your information is located in the database file, by default - "webapp/bseis.sqlite". You can copy the database file to any other place (closed from public access) by using a file manager. The keystore for encryption is located in the "ks" folder. This keystore is protected by a strong password (which you also create). You must use a strong password for authorization in your operating system. It would be better, when you set a password for starting your computer. You can not use the A-Jetty SSL (HTTPS) certificate for any Internet resource (site or WEB-service).

Beigesoft™ EIS on embedded A-Jetty for Android.

Your information (SQLite database), logStd.log, logSec.log, keystore are in the place that is inaccessible to other software. If you forget the keystore or the application password, then you will lose your information. Therefore, you need to periodically make a copy of the database into Google Drive. In addition, you can use encryption for the database, but this is appropriate when you send an encrypted database into Beigesoft™ EIS that is installed on a computer in protected from light harmful software way, or for sharing the database between Android devices via the shared storage (not via Google Drive). Never use the shared storage for exporting or importing non-encrypted private information.

This application has no direct access to your files on Google Drive, that is it can not list or read them. It only asks Google Drive Application for sending or receiving a file interactively, that is with user (your) confirmation.

PDF reports will be downloaded into the Downloads shared folder, so avoid installing unnecessary applications that require access to the storage. When you connect your Android device to a computer in file sharing mode, then any application on the computer will have direct access to these files. You need to delete these PDF files from the shared storage after viewing them.

When all Android applications will be forced to use "scoped storage", then they will have only interactive access to files in the Downloads folder.

When your browser will pass a PDF file to a PDF viewer via the secure transport (not via shared Download folder), then all other applications will have no access to your PDF files.

This application has no direct access to files on the shared storage. You can use the shared storage (just choose it instead of Google Drive) for exporting and importing non-private information (such as ajetty-ca#.pem) or encrypted private one.

You also need to set a screen lock, such as a PIN code.

Beigesoft™ EIS cloud version.

Protecting data published on the Internet is a difficult task. You should use this version only if necessary. You yourself must ensure the reliability of authorization (passwords) and encryption (HTTPS connection) of the transmitted data.
